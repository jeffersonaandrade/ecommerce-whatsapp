# Core v1.1.0 — Multi-client hardening

**Data:** 2026-07-01  
**Tag sugerida:** `v1.1.0`

---

## Resumo

Hardening do core para N deploys independentes, sem fork e sem branching por slug:

- Defaults de identidade neutros (`settings-defaults`, preset plataforma)
- Footer e categorias vitrine somente via banco (sem `siteConfig.categories`)
- `npm run create-client -- <slug>` — scaffold em `deploy/clients/`
- `npm run branding:sync -- --client <slug>`
- `npm run qa:check-no-client-branching` — guard CI anti-slug
- Scaffold SportWear (`deploy/clients/sportwear/`) em draft
- Smoke Playwright ampliado (hero, footer, branding, comercial)

---

## Migrations obrigatórias

**Nenhuma migration nova** nesta release. Compatível com schema até `20260627210000_store_onboarding`.

---

## Breaking changes

Nenhum breaking de schema. Mudança de **comportamento** apenas para lojas sem dados:

- Hero sem imagem no Storage/admin não usa mais fallback Unsplash do core
- Footer sem categorias no banco não exibe coluna "Produtos" (antes: fallback esportivo fixo)

Lojas com `store_settings` e categorias preenchidos (ex.: UnitSports) não são afetadas.
